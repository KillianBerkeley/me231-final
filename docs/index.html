Can We Predict
Employee Burnout?

We trained and compared five different machine learning models — Neural Network, Linear SVM, Gradient Boosted Trees, K-Means Clustering, and Linear Regression — on a 150,000-employee tech-industry mental health dataset to find out which approach best predicts burnout level and what the data can actually tell us.

150,000 employee records
25 input features
5 model types tested
93.4% best accuracy (SVM)
R² 73.9% regression score

🎯 Problem Statement

What are we trying to predict, and why does it matter?

Goal: Given a set of measurable attributes about an employee — their work hours, stress levels, sleep, job satisfaction, and more — can a machine learning model accurately predict whether that employee is experiencing Low, Moderate, or High burnout?

Why Burnout?

Burnout is a state of chronic workplace stress that leads to physical and emotional exhaustion, cynicism, and reduced effectiveness. It is a leading driver of employee turnover, healthcare costs, and lost productivity — especially in the tech industry, where long hours and high pressure are common. Early identification allows organizations to intervene before burnout becomes severe.

The ML Framing

This is a classification problem: given input features, predict which category (Low / Moderate / High) a sample belongs to. We also run a regression task: predicting the raw continuous burnout score. Both tasks let us understand the data from different angles. We compare five approaches to see which family of model handles this data best — and to understand why.

What we're comparing

  • Supervised classifiers — Neural Network, Linear SVM, Boosted Trees
  • Unsupervised clustering — K-Means (does burnout cluster naturally?)
  • Regression — Linear model for the continuous score
  • Ensemble voting — combining model predictions

The three burnout classes

Low (0) Employee is not significantly burned out. Stress and workload are manageable.
Moderate (1) Employee shows signs of burnout — the most common class in this dataset.
High (2) Employee is severely burned out. Immediate intervention likely needed.

Key principle — train/test separation: All five models are trained only on the 80% training set, then evaluated on the 20% test set that they have never seen. This simulates how the model would perform on new, real-world employees. Metrics computed on data used for training would be misleading (overfitting). The scaler is also fit only on training data to prevent data leakage.

Why five different models?

Different model families make fundamentally different assumptions about how the data is structured. By comparing all five, we learn not just what performs best, but why — for example, if a simple linear model matches a deep network, the data's patterns are likely linear. If clustering fails completely, the burnout classes don't form natural geometric groups.

Consistent evaluation

Every supervised classifier uses the exact same train/test split (random seed 42, 80/20). This is critical: if models used different data, you couldn't fairly compare their results. The preprocessing pipeline is shared via a common data_cleaning.py module run once and saved as a bundle.

What do the metrics mean?

Understanding what we're measuring is essential for interpreting the results.

Accuracy
Fraction of all predictions that were correct. Easy to understand, but misleading when classes are imbalanced — a model that always says "Moderate" can get high accuracy by luck.
Precision (weighted)
When the model predicts a class, how often is it right? Weighted means each class is weighted by how many samples it has, so the dominant class matters more.
Recall (weighted)
Of all the employees that actually belong to a class, what fraction did the model correctly identify? Low recall on "High" burnout means real high-burnout cases are being missed.
F1 Score (weighted)
Harmonic mean of precision and recall. Balances both concerns in one number. Weighted version is dominated by the most common class (Moderate).
Macro F1
Average F1 across all classes, giving each class equal weight regardless of size. This is the fairest metric here — it penalizes a model that performs well on Moderate but poorly on Low/High.
R² Score
For regression only. Fraction of variance in the target explained by the model. R²=1.0 is perfect; R²=0 means the model does no better than predicting the mean. Our regression achieves R²=0.74.
Adjusted Rand Index (ARI)
For K-Means only. Measures how well the unsupervised clusters match the true labels (0=random, 1=perfect). Our ARI=0.076 indicates near-random clustering — K-Means essentially failed.
Confusion Matrix
A table showing where the model's predictions went right and wrong: rows are true classes, columns are predicted classes. Diagonal = correct; off-diagonal = errors.

K-Means Clustering

Unsupervised baseline · k=3 · n_init=20

K-Means is an unsupervised algorithm — it has no access to the true burnout labels during training. It groups employees into 3 clusters based purely on feature similarity, then we map each cluster to the majority burnout level within it. Linear Regression

Predicts continuous burnout_score, not the class label

Instead of predicting Low/Moderate/High directly, linear regression predicts the raw numerical burnout score. This is a different task: fitting a weighted sum of all 25 features to minimize prediction error. The score is then converted to classes using percentile cutpoints from the training data (≤33rd percentile → Low, ≤66th → Moderate, else → High).

🤝 Ensemble Methods

An ensemble combines the predictions of multiple models, hoping that their errors are uncorrelated — so where one model is wrong, another is right. We test two voting strategies combining NN, SVM, Boosted Tree, and the binned Regression predictions. K-Means is excluded because its cluster quality is too poor to add signal.

How voting works

For each employee in the test set, each of the four models independently casts a "vote" for the burnout level (Low, Moderate, or High). The final prediction is whichever class gets the most votes. Ties are broken by choosing the lower class index (Low preferred over Moderate, Moderate over High) — a conservative choice.

Equal vs. weighted voting

In equal-weight voting each model gets 1 vote. In weighted voting, each model's vote carries a weight proportional to its test-set macro F1 score — better models have louder voices. In practice, both strategies produced identical results here because the weight differences were small enough to not change any majority decisions.

Why didn't the ensemble beat the best individual model? The SVM achieved 93.40% while both ensembles only reached 93.29%. Including a weaker model (the regression-to-class conversion) diluted the strong signal from the SVM and Boosted Tree. Ensembles work best when the component models make different errors — here, three strong classifiers tend to agree on the same mistakes, so a fourth vote from the regression model occasionally overrules a correct majority.

💡 Key Findings

What we learned from training and comparing all five approaches on 150,000 employee records.

  • 🏆
    Linear SVM achieved the highest accuracy (93.4%) — beating the neural network Despite being structurally simpler than the multi-layer neural network or the boosted tree ensemble, the linear SVM topped all classifiers. This strongly suggests that after StandardScaler normalization, the three burnout classes are nearly linearly separable in the 25-feature space — meaning a flat decision boundary can distinguish Low from Moderate from High without needing the complex curved boundaries that neural networks or boosted trees provide.
  • 🌳
    Boosted Tree placed second (92.84%) despite not needing feature scaling HistGradientBoostingClassifier is inherently scale-invariant — it uses thresholds on raw feature values, not distances. Yet it still came within 0.6% of the SVM. With more hyperparameter tuning or a larger tree depth, it could potentially match the SVM. Its slight edge over the neural network suggests the tree-based splits capture some structure the MLP missed.
  • 🧠
    Neural Network came third — more complexity did not help here The MLPClassifier (64, 32 hidden layers) achieved 91.78%, roughly 1.6% below the SVM. This underperformance suggests the added complexity of multiple non-linear layers is not warranted for this dataset. The patterns that matter for predicting burnout appear to be approximately linear once features are standardized. A deeper or wider network might close the gap, but it would also require far more tuning.
  • 🔵
    K-Means failed completely — burnout does not cluster naturally (ARI = 0.076) An ARI near zero means the unsupervised clusters are almost entirely random with respect to the true burnout labels. The silhouette score of 0.045 (well below the 0.5 threshold for meaningful clusters) confirms that the three burnout classes do not form distinct geometric groupings in feature space. K-Means all but predicted the same cluster for every employee, confirming it cannot distinguish Low from Moderate from High without label supervision.
  • 📈
    Linear Regression explains 73.9% of continuous burnout score variance (R² = 0.74) Even though regression predicts a continuous score rather than a class, achieving R²=0.74 means the 25 input features have meaningful linear relationships with burnout severity. The remaining 26.1% of unexplained variance likely reflects non-linear interactions or unmeasured factors (e.g., personality, management style). The scatter plot shows a well-correlated trend, but systematic over/under-prediction near the extremes confirms the limits of a purely linear fit.
  • 🤝
    Ensemble voting did not beat the best individual model (93.29% vs 93.40%) Both ensemble variants fell 0.11% short of the SVM. This is a common outcome when the component models are high-accuracy and make correlated errors — the "wisdom of crowds" effect breaks down when the crowd agrees on the same mistakes. Including the regression model (with its different class boundaries) added some noise rather than complementary signal.
  • ⚖️
    Class imbalance causes a large gap between weighted F1 (~93%) and macro F1 (~56%) The "Moderate" burnout class is heavily over-represented in the dataset, so all models learn to predict it well. Weighted F1 is pulled up by this dominant class. Macro F1 treats all three classes equally — and here it reveals that "Low" and especially "High" burnout are much harder to classify correctly. In a real deployment, this matters most: failing to identify High burnout cases could have serious consequences. Future work should consider SMOTE resampling, class-weighted losses, or threshold tuning to improve minority-class recall.
